# Аудит входа в систему и Event Viewer (Windows)

## Цель
Включение аудита входа в систему Windows и анализ первых событий
аутентификации в журнале безопасности (Security).

---

## Включение аудита входа

Аудит входа в систему был включён через Local Security Policy
(secpol.msc) с фиксацией успешных и неуспешных попыток входа.

Результаты:
- secpol_opened.png
- audit_policy_section.png
- audit_logon_events_enabled.png
- audit_account_logon_enabled.png

---

## Генерация событий входа

Для создания событий были выполнены:
- перезагрузка системы;
- успешный вход под администратором;
- блокировка и повторный вход;
- намеренно введён неправильный пароль.

---

## Анализ журнала безопасности

Журнал событий был открыт через Event Viewer (eventvwr.msc).

Путь:
Журналы Windows → Безопасность

Проанализированы:
- дата и время событий;
- идентификаторы событий;
- типы входа;
- успешные и неуспешные попытки.

Результаты:
- event_viewer_opened.png
- security_log_opened.png
- security_log_overview.png
- logon_event_details.png
- special-logon-4672
- failed-logon-4625


---

## Выводы
- Аудит входа является базовым механизмом контроля доступа.
- Все попытки входа фиксируются в журнале Security.
- Журналы событий являются основной доказательной базой
  при расследовании инцидентов информационной безопасности.
